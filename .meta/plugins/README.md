# Plugins Directory

This directory is for **plugins** - modular capabilities you create for your toolkits.

---

## What Goes Here

After you run `/start-here` and create your first toolkit, come back here and add plugins to this folder. They will be available for selection on every future toolkit generation.

**Examples of what plugins might be**:
- Custom processors for specific file types
- Specialized analysis capabilities
- Domain-specific transformations
- Reusable AI prompt sets

---

## Creating Your First Plugin

When you're ready to create a reusable plugin:

1. See `.meta/PLUGIN_GUIDE.md` for detailed instructions
2. Create `.meta/plugins/your-plugin-name/`
3. Add README, CONFIG, and any code/prompts
4. `/start-here` will detect it automatically on the next toolkit generation

---

## This Directory is Currently Empty

That's intentional. Build your plugins as you discover patterns worth reusing.

**Workflow**:
1. Generate toolkit → use it → discover reusable pattern
2. Extract pattern into plugin
3. Add to this directory
4. Reuse in future toolkits

---

**Start with your first toolkit. Plugins come later when you have patterns to extract.**
